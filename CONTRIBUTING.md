# 📘 Guía de Contribución - OmniSens

## Flujo de trabajo

1. Crear Issue
2. Asignar labels y milestone
3. Crear rama (branch)
4. Desarrollar tarea
5. Crear Pull Request
6. Revisar y aprobar

---

## Reglas

- No subir credenciales
- Documentar cambios
- Usar nombres claros
- Trabajar siempre con Issues

---

## Convención de ramas

- feature/nombre
- fix/nombre
- docs/nombre

---

## Definition of Done

Una tarea se considera finalizada cuando:

- Está documentada
- Fue revisada
- Está en el repositorio
- No tiene errores
